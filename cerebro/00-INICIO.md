# Cerebro AMAS Team Wolf

> Lee estos archivos en orden numérico para entender todo el proyecto.

## Orden de lectura

1. `01-proyecto.md` — Qué es, quién lo usa, stack técnico
2. `02-arquitectura.md` — Frontend, backend, BD, deploy, servicios externos
3. `03-rutas-endpoints.md` — Todas las rutas del frontend y API
4. `04-base-datos.md` — Tablas, funciones SQL, vistas
5. `05-horarios-asistencia.md` — Sistema QR, horarios por día, panel profesora
6. `06-decisiones.md` — Decisiones técnicas tomadas y por qué
7. `07-mejoras-realizadas.md` — Historial de optimizaciones (fases 1-10)
8. `08-problemas-conocidos.md` — Lo que falta, bugs pendientes, limitaciones
9. `09-deploy.md` — Cómo desplegar frontend y backend
10. `10-accesos.md` — Referencia a archivo de accesos (no en git)

## Cómo usar este cerebro

Cuando abras una nueva terminal de Claude Code en este proyecto:
1. Lee `CLAUDE.md` primero (reglas del proyecto)
2. Lee `cerebro/00-INICIO.md` (este archivo)
3. Lee los archivos del cerebro que necesites según la tarea
4. No necesitas leer todos — usa el índice de arriba para ir al tema relevante
